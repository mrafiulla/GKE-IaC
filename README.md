data-anchorjs-icon="" style="font-style: normal; font-variant: normal; font-kerning: auto; font-optical-sizing: auto; font-feature-settings: normal; font-variation-settings: normal; font-weight: normal; font-stretch: normal; font-size: 1em; line-height: inherit; font-family: anchorjs-icons; position: absolute; margin-left: -1em; padding-right: 0.5em;"></a>What is it?</h3><p>A<!-- --> <a href="https://en.wikipedia.org/wiki/README">README</a> <!-- -->is a text file that introduces and explains a project. It contains information that is commonly required to understand what the project is about.</p></div><div class="mb-8 mx-4"><h3 id="why-should-i-make-it"><a class="anchorjs-link " href="https://www.makeareadme.com/#why-should-i-make-it" aria-label="Anchor" data-anchorjs-icon="" style="font-style: normal; font-family: anchorjs-icons; position: absolute; margin-left: -1em; padding-right: 0.5em;"></a>Why should I make it?</h3><p>It's an easy way to answer questions that your audience will likely have regarding how to install and use your project and also how to collaborate with you.</p></div><div class="mb-8 mx-4"><h3 id="who-should-make-it"><a class="anchorjs-link " href="https://www.makeareadme.com/#who-should-make-it" aria-label="Anchor" data-anchorjs-icon="" style="font-style: normal; font-family: anchorjs-icons; position: absolute; margin-left: -1em; padding-right: 0.5em;"></a>Who should make it?</h3><p>Anyone who is working on a programming project, especially if you want others to use it or contribute.</p></div><div class="mb-8 mx-4"><h3 id="when-should-i-make-it"><a class="anchorjs-link " href="https://www.makeareadme.com/#when-should-i-make-it" aria-label="Anchor" data-anchorjs-icon="" style="font-style: normal; font-family: anchorjs-icons; position: absolute; margin-left: -1em; padding-right: 0.5em;"></a>When should I make it?</h3><p>Definitely before you show a project to other people or make it public. You might want to get into the habit of making it the<!-- --> <a href="http://tom.preston-werner.com/2010/08/23/readme-driven-development.html">first file you create</a> <!-- -->in a new project.</p></div><div class="mb-8 mx-4"><h3 id="where-should-i-put-it"><a class="anchorjs-link " href="https://www.makeareadme.com/#where-should-i-put-it" aria-label="Anchor" data-anchorjs-icon="" style="font-style: normal; font-variant: normal; font-kerning: auto; font-optical-sizing: auto; font-feature-settings: normal; font-variation-settings: normal; font-weight: normal; font-stretch: normal; font-size: 1em; line-height: inherit; font-family: anchorjs-icons; position: absolute; margin-left: -1em; padding-right: 0.5em;"></a>Where should I put it?</h3><p>In the top level directory of the project. This is where someone who is new to your project will start out. Code hosting services such as<!-- --> <a href="https://github.com/">GitHub</a>,<!-- --> <a href="https://bitbucket.org/">Bitbucket</a>, and<!-- --> <a href="https://about.gitlab.com/">GitLab</a> will also look for your README and display it along with the list of files and directories in your project.</p></div><div class="mb-8 mx-4"><h3 id="how-should-i-make-it"><a class="anchorjs-link " href="https://www.makeareadme.com/#how-should-i-make-it" aria-label="Anchor" data-anchorjs-icon="" style="font-style: normal; font-variant: normal; font-kerning: auto; font-optical-sizing: auto; font-feature-settings: normal; font-variation-settings: normal; font-weight: normal; font-stretch: normal; font-size: 1em; line-height: inherit; font-family: anchorjs-icons; position: absolute; margin-left: -1em; padding-right: 0.5em;"></a>How should I make it?</h3><p>While READMEs can be written in any text file format, the most common one that is used nowadays is<!-- --> <a href="https://en.wikipedia.org/wiki/Markdown">Markdown</a>.
